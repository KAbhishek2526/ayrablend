### Architecture Overview
**Frontend (React + Vite + Tailwind CSS)**
- Single Page Application with client-side routing
- State management using React Context API
- Persistent cart storage using localStorage
- Responsive design with Tailwind CSS

**Backend (Node.js + Express)**
- RESTful API architecture
- MVC pattern implementation
- JWT-based authentication (optional)
- Rate limiting and security middleware
**Database (MongoDB Atlas)**
- Cloud-hosted NoSQL database
- Indexed collections for performance
- Automatic backups and scaling
**Third-Party Services**
- **Cloudinary**: Image optimization and CDN delivery
- **Razorpay**: Primary payment gateway with UPI/Cards/
NetBanking
- **PayPal**: Optional secondary payment method
- **WhatsApp Cloud API**: Order fallback and customer 
communication

## 3. API Endpoints
### Products API

| Method | Endpoint | Description | Auth Required |
|--------|----------|-------------|---------------|
| GET | `/api/products` | Get all products with pagination & п¬Ғlters | 
No |
| GET | `/api/products/:id` | Get single product by ID | No |
| GET | `/api/products/slug/:slug` | Get product by slug | No |
| GET | `/api/products/category/:category` | Get products by 
category | No |
| POST | `/api/products` | Create new product | Yes (Admin) |
| PUT | `/api/products/:id` | Update product | Yes (Admin) |
| DELETE | `/api/products/:id` | Delete product | Yes (Admin) |
**Query Parameters for GET /api/products:**
- `page` (number, default: 1) - Pagination page
- `limit` (number, default: 12) - Items per page
- `category` (string) - Filter by category
- `minPrice` (number) - Minimum price п¬Ғlter
- `maxPrice` (number) - Maximum price п¬Ғlter
- `sort` (string) - Sort by: 'price_asc', 'price_desc', 'newest', 'popular'
- `search` (string) - Search in product name/description
### Orders API
| Method | Endpoint | Description | Auth Required |
|--------|----------|-------------|---------------|
| POST | `/api/orders` | Create new order | No |
| GET | `/api/orders/:id` | Get order by ID | Yes (Owner/Admin) |
| GET | `/api/orders/phone/:phone` | Get orders by phone number | 
Yes (Owner) |
| PUT | `/api/orders/:id/status` | Update order status | Yes (Admin) |
| POST | `/api/orders/:id/whatsapp` | Send WhatsApp fallback | No |
| GET | `/api/orders/track/:orderId` | Track order status | No |
### Payment API
| Method | Endpoint | Description |
|--------|----------|-------------|

| POST | `/api/payment/create-order` | Create Razorpay order |
| POST | `/api/payment/verify` | Verify payment signature |
| POST | `/api/payment/webhook` | Razorpay webhook endpoint |
| POST | `/api/payment/paypal/create` | Create PayPal order |
| POST | `/api/payment/paypal/capture` | Capture PayPal payment |
